네이버페이, 토스페이, 페이팔은 결제 방식, 연동 과정, 수수료, 사업자 등록 필요 여부에서 각각 차이가 있습니다. 아래에 수수료와 필수 사업자 등록 요건, 결제 로직 구현 팁을 정리했습니다.

## 결제 API 차이점

- **네이버페이**: 쇼핑몰 내에서 바로 결제 가능하며, 네이버 계정만 있으면 이용 가능. 결제 진행이 매끄럽고 사용자가 별도 앱을 이동하지 않아도 됨.
- **토스페이**: 결제 과정에서 토스앱으로 이동하여 비밀번호 등 인증 후 완료됨. 일부 쇼핑몰은 결제 버튼 클릭 후 토스앱을 통해 인증 절차 진행. API 연동 후 토스페이먼츠를 통해 Paypal도 연동 가능.
- **PayPal**: 해외 결제에 최적화, 바로 PayPal 결제 페이지로 이동해 진행. 국내에서 토스페이먼츠가 연동 중개 역할 가능. PayPal 계정 필요.

### 토스페이 VS 토스페이먼츠

둘은 서로 다른 개념이다. 토스페이는 토스 앱에서 제공하는 간편결제 서비스이며, 토스 계정과 연동된 카드 또는 계좌로 빠르게 결제 가능,
토스페이먼츠는 온라인 사업가를 대상으로 한 전자결제대행 서비스, 사업자가 카드, 간편결제 등 다양한 결제수단을 한 번에 연동해서 받을 수 있도록 지원

## 사업자등록 필요 여부

- **실서비스(수익 발생)**: 네이버페이, 토스페이, 페이팔 모두 사업자 등록 필요. 결제사 API(토스페이, 네이버페이) 이용 시 사업자 등록을 요구함.
- **테스트 목적(수익/실거래 없음)**: 사업자등록 없이도 테스트 API·키를 제공하는 경우가 많음. 토스페이먼츠와 네이버페이는 개발자센터 테스트 모드에서 사업자등록 없이 개발·테스트 가능.
- 페이팔 개발자 계정도 사업자 정보 없이 연동·테스트 가능.

## 결제 로직 구현 팁

- 쇼핑몰 주문서(Checkout) 페이지에서 결제방식(네이버페이, 토스페이, 페이팔) 선택 UI 구현
- 각 결제 수단별 API/SDK 연동: 주문정보와 사용자 정보로 결제창(또는 결제 요청) 생성
  - 네이버페이 API: 플랫폼 내 결제창과 연동
  - 토스페이 API: 결제요청 시 토스앱 이동/인증, 웹훅 등 결제완료 이벤트 처리
  - 페이팔: PayPal SDK/REST API로 결제 요청, 결과 콜백 처리
- 결제 완료 시, PG사/페이팔의 성공/실패 webhook 또는 리턴값으로 주문내역 업데이트
- 테스트 환경은 각 사의 개발자센터에서 제공하는 키, 샌드박스 모드 이용

결제 흐름에 따라 공통적으로 주문 → 결제수단 선택 → 각 사의 결제 API 요청 → 결제완료 콜백/웹훅 → 주문 처리로 구현하면현하면 됩니다.
