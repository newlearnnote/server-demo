**이주철 - NoteBookmarkModule**

1. 본인이 북마크한 노트 조회할 때, 반환 타입 NoteDetail[]로 변경 -> 변경완료
2. 노트 북마크 soft delete로 변경 -> update로 변경
3. 노트 북마크 테스트 -> 진행중
4. 노트 북마크 리팩토링 -> 완료

**원도현 - NoteModule**

5. 노트 publish 할 때, 이미 한 적이 있다면 덮어씌우기.(현재는 GCS에서 덮어지지만, db에는 새로 만들어짐) - create이자 update 임.
6. 노트 publish할 때, 이미 publish 된 적 있는 노트라면, publish가 아닌 create가 아닌 update임. 업데이트할 때 옛날 태그 연결 softDelete

```
tagId, noteId
1, 2 --> deletedAt
2, 2 --> deletedAt
3, 2 -- 새로 연결된 태그들
4, 2 -- 새로 연결된 태그들
```

7. Note publish dto 변경. network할 노트들 전달

> DESKTOP APP
>
> 1. 북마크 API 연동
> 2. 노트 페이지 UI 변경

> 4. publish 할 때 연결된 노트 id 전달

> WEB
>
> 1. 북마크 API 연동
